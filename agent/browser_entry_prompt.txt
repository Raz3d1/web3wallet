【任务】从上述元素中判断：
1. **主界面 (main)**：钱包 App 首页/资产页。选择要点击的元素时按以下优先逻辑：
   - **优先**：若有明显的「浏览器」按钮或图标（如地球/globe 图标、DApp、探索、Discover、应用、网页等），直接选该按钮/图标的 id。
   - **否则**：若没有明显图标，但有输入框且其文案或 placeholder 含 search、URL、网址、搜索 等字样，则选该输入框的 id（点击后可输入网址）。
2. **地址栏已出现 (address_bar)**：已经进入内置浏览器，界面中有明显的 URL 输入框、地址栏、搜索框。
   → 请给出该地址栏/输入框元素的 id。
3. **测试页已加载 (dapp_loaded)**：页面中已经出现「Web3Wallet」「自动化测试」「Web3 自动化测试」或一列带「▶」的测试按钮（如「危险的 eth_sign」「ERC20 授权」等），说明测试 Dapp 已打开。
   → 无需点击，直接返回 dapp_loaded。
4. **访问链接确认 (visit_link)**：输入地址并回车后，界面没有直接跳转，而是出现一个带目标网址的「Visit link」「打开链接」「前往」「Go」「访问」等可点击项，需要用户点击该项才会真正打开网页。
   → 请给出该可点击项（按钮/链接）的 id，以便脚本点击进入。
5. **未知 (unknown)**：无法判断，或既不是主界面也没有地址栏、也不是测试页、也不是访问链接确认页。

【返回 JSON（仅一种情况）】
{
  "screen_type": "main 或 address_bar 或 dapp_loaded 或 visit_link 或 unknown",
  "element_id": 数字或 null
}

规则：
- screen_type 为 main 时，element_id 为「浏览器/DApp/探索」入口的 id。
- screen_type 为 address_bar 时，element_id 为地址栏输入框的 id。
- screen_type 为 visit_link 时，element_id 为「Visit link/打开链接/前往」等确认项的 id。
- screen_type 为 dapp_loaded 或 unknown 时，element_id 为 null。
